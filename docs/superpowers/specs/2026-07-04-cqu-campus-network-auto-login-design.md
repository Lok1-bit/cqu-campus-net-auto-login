# 重庆大学校园网自动登录设计

## 目标

在 Windows 用户登录后静默启动监控程序。程序定期检查重庆大学 Dr.COM 校园网认证状态；掉线时自动重新认证，不干扰正常网络使用。

## 已确认的门户协议

- 门户入口：`https://login.cqu.edu.cn/`
- 产品：Dr.COM
- 在线状态接口：`/drcom/chkstatus`
- 普通账号登录接口：`/drcom/login`
- 账号与密码字段：`DDDDD`、`upass`
- 当前门户配置不启用密码 MD5；请求仍必须通过 HTTPS 发送。

## 方案选择

采用纯 PowerShell 实现，避免安装 Python 或第三方依赖。通过 Windows 凭据管理器保存账号密码，通过 Windows 计划任务在用户登录后静默启动。

不采用浏览器自动化，因为其资源占用更高，且容易受页面布局变化影响。不采用 Python，是为了降低安装和维护成本。

## 文件结构

- `cqu-campus-net.ps1`：状态检测、自动登录、退避和日志的主程序。
- `config.psd1`：可编辑配置，默认在线复查间隔为 30 秒。
- `install.ps1`：收集凭据、写入 Windows 凭据管理器并创建计划任务。
- `uninstall.ps1`：删除计划任务和对应凭据；保留日志供用户自行检查或删除。
- `README.md`：安装、配置、运行、排错和卸载说明。
- `tests/`：无需真实账号的协议解析与控制逻辑测试。

## 配置

`config.psd1` 至少提供以下设置：

- `CheckIntervalSeconds = 30`：在线时的复查周期，可由用户修改。
- `FailureBackoffSeconds = @(10, 30, 60, 120, 300)`：门户不可达或登录失败时的退避序列。
- `RequestTimeoutSeconds = 10`：单次 HTTP 请求超时。
- `LogMaxBytes = 1048576`：单个日志文件的最大大小。

主程序启动时校验配置。无效值使用安全默认值并写入日志，但不会中止整个监控程序。

## 运行流程

1. 计划任务在当前用户登录后启动隐藏的 PowerShell 进程。
2. 主程序调用 `chkstatus` 并解析 JSON 或 JSONP 响应。
3. 若已经在线，等待 `CheckIntervalSeconds` 后复查。
4. 若门户可达但未认证，从响应中取得当前 IPv4/IPv6，再从 Windows 凭据管理器读取凭据。
5. 主程序通过 HTTPS 调用 `drcom/login`，提交门户要求的账号、密码、IP、终端类型和固定协议参数。
6. 登录请求结束后再次调用 `chkstatus`；仅以复查结果作为成功依据。
7. 若门户不可达，视为网络未就绪或设备不在重大校园网，不发送账号密码，并按退避序列重试。

任何时刻最多只有一个状态请求或登录请求，不进行并发访问。

## 凭据与安全

- 安装脚本使用安全密码输入框，不回显密码。
- 凭据以固定目标名保存在当前用户的 Windows 凭据管理器中。
- 主程序仅在确认重大门户可达且返回未认证状态后读取凭据。
- 密码只发送到 `login.cqu.edu.cn` 的 HTTPS 登录接口。
- 日志不记录账号、密码、完整查询字符串、姓名、MAC 地址或完整 IP 地址。
- 错误信息先清理潜在敏感字段并限制长度后再写入日志。

## 错误处理与资源控制

程序区分三种状态：已在线、门户可达但未认证、门户不可达。

- 已在线：低频轮询，不发登录请求。
- 未认证：尝试一次登录并复查；失败后进入退避。
- 门户不可达：不读取或发送凭据，直接退避。
- 相同错误进行限频记录，避免日志快速增长。
- 日志达到大小限制时滚动保留一个旧文件。
- 计划任务禁止并行启动第二个实例；进程异常退出时由任务计划程序重启。

常驻 PowerShell 预计占用约 40–80 MB 内存，空闲 CPU 接近零。每次检查只产生一个小型 HTTP 请求，不会造成可感知的网络卡顿。

## 安装与卸载

安装脚本执行以下操作：

1. 检查 Windows PowerShell 版本和脚本文件完整性。
2. 提示用户输入账号密码并写入凭据管理器。
3. 创建仅对当前用户生效的登录触发计划任务。
4. 以隐藏窗口启动主脚本，并设置失败后自动重启。
5. 执行一次不提交密码的状态检查，报告安装结果。

卸载脚本停止并删除计划任务，然后删除对应凭据。卸载不修改网络设置、不注销当前校园网会话。

## 测试策略

将 JSONP 解析、在线状态判断、登录参数生成、配置校验、退避选择和日志脱敏实现为独立函数。

自动测试覆盖：

- 已在线、未在线、格式错误和超时响应。
- IPv4、IPv6 以及缺少地址的响应。
- 参数 URL 编码及必需字段。
- 配置边界值与退避上限。
- 日志中不得出现模拟账号、密码、MAC 或完整 IP。

测试不调用真实登录接口，不需要真实凭据。真实联调由用户明确运行一次测试命令完成；实现阶段不会自动使用用户账号提交登录请求。

## 成功标准

- Windows 用户登录后无需打开窗口即可启动监控。
- 在线时不重复认证，掉线后能自动恢复。
- 不在校园网时不提交凭据，也不高频重试。
- 检查周期可通过 `config.psd1` 修改。
- 密码不以明文文件保存，日志不泄露敏感信息。
- 安装和卸载均有清晰、可重复的命令。
