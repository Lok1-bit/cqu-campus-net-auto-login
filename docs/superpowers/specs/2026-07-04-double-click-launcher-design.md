# 双击图形启动器设计

## 目标

让普通 Windows 用户无需打开 PowerShell 或输入命令，通过双击一个文件完成安装、状态检查、凭据更新和卸载，同时保留现有命令行接口。

## 方案

采用一个 CMD 双击入口和一个 PowerShell WinForms 界面：

- `CQU校园网工具.cmd`：用户唯一需要双击的入口。它使用 `powershell.exe -NoProfile -ExecutionPolicy Bypass -WindowStyle Hidden` 启动图形界面，然后立即关闭批处理窗口。
- `launcher.ps1`：创建固定大小的 Windows Forms 窗口，显示项目状态和四个操作按钮。

不打包 EXE，避免增加编译、签名和二进制信任问题。不采用黑色控制台菜单，因为目标是让非技术用户完全不接触命令行。

## 界面与操作

窗口标题为 `CQU 校园网自动登录`，正文明确标注“非重庆大学官方项目”。界面提供：

1. **安装并立即启动**：调用 `install.ps1 -StartNow`。Windows 安全凭据框收集账号和密码；成功后显示提示。
2. **检查当前状态**：直接使用模块的 `Invoke-CquStatusRequest`，显示“已在线”“需要认证”或“门户不可达”，不读取凭据。
3. **更新账号密码**：复用安装流程覆盖 Windows 凭据管理器中的记录并保持计划任务有效。
4. **卸载**：先显示确认框，确认后调用 `uninstall.ps1`；默认保留日志。

窗口底部显示计划任务是否存在、后台任务当前状态，以及最后一次操作结果。提供“打开日志目录”和“关闭”辅助按钮。

## 调用边界

状态检查在界面进程内调用现有模块，避免启动控制台。会修改系统状态的安装、更新和卸载通过独立的隐藏 PowerShell 子进程调用现有脚本，避免复制凭据与计划任务逻辑。

启动器不接收、解析或保存密码。凭据仍由 `Get-Credential` 安全窗口获取，并由 `install.ps1` 写入 Windows 凭据管理器。

## 错误处理

- 缺少核心脚本时，显示缺失文件名并禁止相关操作。
- 子进程返回非零退出码时，显示简短错误并引导查看日志或重新运行。
- 状态请求超时或门户不可达时，只显示网络提示，不读取或发送凭据。
- 重复点击操作按钮时暂时禁用全部操作按钮，防止并行安装或卸载。
- 卸载必须二次确认；取消时不产生任何系统变更。

## 兼容性

- 目标系统为 Windows 10/11 与 Windows PowerShell 5.1。
- WinForms 使用系统自带的 `System.Windows.Forms` 和 `System.Drawing`，不增加依赖。
- CMD 入口通过 `%~dp0` 定位脚本，支持项目位于含空格或中文的目录。
- `.gitattributes` 为 `.cmd` 指定 CRLF，避免双击执行时的换行问题。

## 测试

自动测试不打开窗口、不提交凭据、不创建计划任务：

- PowerShell 解析器检查 `launcher.ps1`。
- 静态测试确认 CMD 入口只调用仓库内的 `launcher.ps1`，使用执行策略绕过和隐藏窗口参数。
- 静态测试确认四个操作与现有脚本/模块连接，状态检查不会调用登录函数。
- CI 继续运行全部离线 Pester 测试。

人工验收仅检查界面能打开、按钮文字正确和取消凭据输入不会安装；不要求提供真实账号。

## 验收标准

- 从 GitHub ZIP 解压后可直接双击打开图形界面。
- 普通用户无需输入命令即可完成四项核心操作。
- 无控制台窗口常驻或闪烁。
- 凭据存储、安全边界和后台监控行为与现有版本一致。
- 本地与 GitHub CI 测试通过，`v0.1.1` 标签指向通过 CI 的提交。
